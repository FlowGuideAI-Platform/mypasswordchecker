Frequently Asked Questions

Is it safe to test my real password here?

Technically yes, but we recommend testing a similar password instead. All analysis happens 100% in your browser - your password never leaves your device. However, as a best practice, test a password with a similar pattern rather than your actual password. This protects you from keyloggers, screen recording malware, or shoulder surfing.

How can I verify this is really client-side?

Easy! Here's how to verify:

  1. Open your browser's Developer Tools (press F12 or right-click → Inspect)
  2. Go to the "Network" tab
  3. Type a password in our checker
  4. Watch the Network tab - you'll see zero requests to our servers

This proves your password stays in your browser. You can also view our open-source code on GitHub.

What makes a password "strong"?

A strong password has these characteristics:

  • Length: At least 12-16 characters (longer is better)
  • Character diversity: Mix of uppercase, lowercase, numbers, symbols
  • No patterns: Avoid "password123", "qwerty", birthdays, names
  • High entropy: 60+ bits for classical security, 80+ for quantum resistance
  • Uniqueness: Different password for each account
  • Not breached: Check against known breach databases (available with our Premium or API access)

Can I check if my password has been in a data breach?

Yes! We offer a Password Breach Check feature that uses the Have I Been Pwned (HIBP) database to check if your password appears in over 800 million breached passwords from known data breaches.

How it protects your privacy: Your password is hashed using SHA-1 in your browser, and only the first 5 characters of the hash are sent to our API using k-anonymity. This means 99.9875% of your password hash remains completely private. Neither we nor HIBP ever see your actual password.

This feature requires an API key and is available with our Premium access ($1) or Developer API (free tier available).
